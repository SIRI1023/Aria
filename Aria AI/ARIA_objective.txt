  ---
  What Aria Should Actually Be

  "An AI operations agent that connects to your business tools via MCP, answers questions from live data, executes multi-step tasks, and shows you exactly how it thinks — so you can trust it."

  The three things that make it defensible:

  1. MCP as the integration backbone (Phase 4)
  Instead of hardcoding Notion, Slack, Drive connectors like everyone else — you use the Model Context Protocol (open standard by Anthropic). Any tool with an MCP server plugs in instantly. This is the
  architectural bet that makes Aria extensible without re-engineering.

  Interview talking point: "I built the integration layer on MCP instead of custom connectors — adding a new tool takes hours not weeks, and the agent automatically discovers what tools can do."

  2. Transparent multi-agent reasoning (Phase 3)
  Most AI tools are black boxes. Aria shows the agent's plan, each tool call, and each decision in real-time in the UI. This is a trust feature that enterprise buyers specifically ask for.

  Interview talking point: "I built a streaming agent trace viewer — users see the planner decompose the task, watch each tool execution, and can intervene before the agent takes action."

  3. Action-capable, not just read-only (Phase 3+4)
  Aria doesn't just answer "what are my open Notion tasks?" — it can say "here are your 5 open tasks, want me to prioritize and assign deadlines?" and execute that with approval. Read + write with
  human-in-the-loop.

  Notion (docs/projects) + Slack (conversations) + Google Drive (files)
  + Jira (tickets) + CRM (customers) + Spreadsheets (data)

  - Spend 30 min searching Slack for a decision made 2 weeks ago

  - Miss context because the answer lives in a thread they weren't part of
